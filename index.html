SCM & UI/UX Analysis

A case study on improving a critical government service UI.

The "Ugliest" UI I've Seen: Cagayan de Oro PWD ID Registration

The **PWD ID Online Registration** site is critical for accessibility but suffers from severe usability and information architecture issues. Given the target audience (Persons With Disabilities), the poor design creates significant barriers to access.

How I Would Improve It (Focusing on Accessibility & Clarity)

The priority is to adopt WCAG (Web Content Accessibility Guidelines) standards, simplify the registration flow, and ensure the UI is usable by people with various disabilities.

1. Staged Registration Flow (Usability)

Break the registration process into distinct, titled steps (e.g., 1. Requirements, 2. Personal Data, 3. Upload Documents). Use a progress tracker (stepper) at the top. The current one-page, form-heavy design is overwhelming and confusing.

2. Enhanced Accessibility (WCAG Compliance)

Significantly increase text contrast and font size for legibility (WCAG 2.1 AA+). Ensure all form fields have proper `

3. Clear Error & Validation Messages

Error messages like "The code you entered is not valid" are currently displayed confusingly. These must be contextual, immediately visible next to the failed input, and use ARIA live regions to announce errors to screen readers.

4. Simplified Information Architecture

Move auxiliary information (Vision, Mission, other related systems) to a separate "About" page or footer. Only the PWD Registration information, the form, and clear contact details should occupy the main screen to reduce cognitive load.
