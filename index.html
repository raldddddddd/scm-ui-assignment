SCM & UI/UX Analysis

A case study on improving a notoriously "ugly" UI.

The "Ugliest" UI I've Seen: Arngren.net

A classic example of overwhelming and chaotic design is the Norwegian classifieds site, Arngren.net. It famously violates nearly every modern principle of usability, accessibility, and visual hierarchy.

How I Would Improve It

The core issue is a complete lack of information architecture. My improvement plan would focus on "Progressive Disclosure"â€”only showing the user what they need, when they need it.

1. Structure & Hierarchy

Implement a clear grid-based layout (e.g., 12-column grid). Use a standard navigation bar at the top with clear categories (e.g., "Electronics," "Vehicles," "Hobbies") instead of listing every single item on the homepage.

2. Visual Consolidation

Group related items into "cards." Each card would have a small image, a concise title, a price, and a "View Details" button. This cleans up the visual noise and makes the page scannable.

3. Search and Filtering

The *most* important feature to add would be a prominent search bar at the top of the page. Alongside it, I'd add filters for category, price range, and location to help users narrow down the overwhelming number of products.

4. Branding & Typography

Establish a consistent color palette (e.g., one primary brand color, one accent color, and neutral grays). Use a single, readable font (like this "Inter" font) in 2-3 consistent sizes for headings, body text, and labels.
